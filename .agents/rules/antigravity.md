# 🛡️ Antigravity: Global Coding Standards

Этот файл содержит незыблемые правила написания кода для Antigravity и любых других агентов, работающих над кодовой базой.

## 🚫 The "Zero-Any" Policy

Мы пишем Production-Ready код. TypeScript используется для обеспечения надежности, а не как декорация.

1. **Запрет на `any`**:
   - Использование типа `any` КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО.
   - Если данные динамические или их структура неизвестна на этапе компиляции, используйте `unknown`.
2. **Type Guards**:
   - При использовании `unknown` ОБЯЗАТЕЛЬНО проверяйте тип через `typeof`, `instanceof` или кастомные Type Guards перед использованием данных.
3. **Strict Mocking (Тесты)**:
   - Не используйте `as any` в тестах.
   - Используйте структурную типизацию: `Object.assign(() => {}, { property: value })` или `Pick<Type, 'field'>`.
   - Если мокается внешняя библиотека, найдите её тип или опишите минимально необходимый интерфейс.

## 🏗️ Architecture Rules

1. **Path Aliases**: Все внутренние импорты должны использовать префикс `@/`.
2. **Типизация**: Типы должны храниться в `src/core/` и быть разделены по смыслу. Никаких гигантских `types.ts`.
3. **Services**: Вся логика, которая может потребоваться более чем одной команде, выносится в сервисы (`src/services`). Команды в `src/commands` должны быть максимально тонкими.
