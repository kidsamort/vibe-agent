# Обзор Системы Управления Задачами (Канбан)

_**Цель этого документа:** Объяснить, как устроена наша система отслеживания задач, и служить точкой входа для всех канбан-досок проекта._

---

## 1. Философия: Мульти-доски, Гибридные Эпики и Семантические ID

Мы используем систему из **нескольких, сфокусированных канбан-досок**. Каждая доска предназначена для отдельного потока работ.

Задачи хранятся в централизованном **"Пуле Задач" (`04-tasks-pool/`)**. Колонки на досках (например, `inbox.md`) содержат только **ссылки** на детальные описания задач в этом пуле.

### Группировка Задач: Гибридная Система Эпиков

Для организации связанных задач мы используем прагматичную гибридную систему, которая сочетает простоту и машиночитаемость.

1.  **Справочник Эпиков:** Для каждой доски существует файл `00-epics-list.md`, который является "единым источником правды" для всех эпиков этого потока работ. Каждый эпик в этом файле имеет **уникальный ID** и читаемое название.
2.  **Машиночитаемая Связь:** Каждая задача в своем паспорте (`01-index.md`) **обязана** иметь поле `epic`, в котором указывается ID родительского эпика. Это создает жесткую, машиночитаемую связь, позволяя строить граф знаний.
3.  **Визуальная Группировка:** В файлах-колонках канбан-доски (`inbox.md`, `in-progress.md` и т.д.) задачи визуально группируются под заголовками, соответствующими названиям эпиков для удобства чтения.

Эта система позволяет нам избежать избыточной сложности (отдельных папок для эпиков), сохраняя при этом мощную связь между задачей и ее высокоуровневой целью.

### Система Идентификации Задач

Чтобы гарантировать уникальность каждой задачи, мы используем **семантическую систему префиксов**.

**Формат:** `[ПРЕФИКС]-[НОМЕР]`

Детальное описание префиксов находится в соответствующем разделе документации процесса (`04-process/`).

## 2. Как Создать и Вести Задачу

1.  **Определите ID:** Выберите доску, определите следующий порядковый номер и сформируйте ID (например, `DEV-102`).
2.  **Создайте Папку Задачи:** В директории `04-tasks-pool/` создайте папку (например, `DEV-102-new-feature`).
3.  **Используйте Шаблон:** Скопируйте в нее содержимое из **[шаблона задачи](../03-task-template/)**.
4.  **Заполните Паспорт:** В файле `01-index.md` укажите `id`, `title` и, что самое важное, **заполните поле `epic`**, указав ID родительского эпика из соответствующего `00-epics-list.md`.
5.  **Добавьте Ссылку на Доску:** В соответствующей колонке (например, `01-inbox.md`) добавьте ссылку на паспорт вашей задачи под заголовком соответствующего эпика.
