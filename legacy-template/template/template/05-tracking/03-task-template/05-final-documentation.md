## 1. Краткий Обзор Реализации

_**Инструкция:** В 1-2 абзацах опишите, что было сделано. Какая проблема решена? Какой основной результат был достигнут?_

**Пример:**
> В рамках этой задачи была реализована модульная система аутентификации. Она включает в себя React-компоненты для форм, хуки для управления состоянием и сервис для взаимодействия с API. Система использует JWT-токены, которые хранятся в `localStorage`, и автоматически обновляет их при истечении срока действия.

## 2. Итоговая Архитектура и Логика

_**Инструкция:** Опишите, как устроена реализованная фича. Какие новые файлы/модули/компоненты были созданы? Как они взаимодействуют друг с другом? Можно использовать небольшие диаграммы или схемы._

**Пример:**
> ### Компоненты
> - **`AuthForm.tsx`**: "Глупый" компонент, отвечающий за рендеринг полей ввода и кнопки.
> - **`LoginScreen.tsx`**: "Умный" компонент, который использует `useAuth` и управляет процессом входа.
>
> ### Хук `useAuth`
> - Содержит всю логику: отправка запросов, обработка ошибок, сохранение токена.
>
> ### Сервис `AuthService`
> - Инкапсулирует вызовы `fetch` к эндпоинтам `/login` и `/register`.

## 3. Как Использовать / Руководство по Эксплуатации

_**Инструкция:** Если ваша задача добавила что-то, что могут использовать другие разработчики (новый компонент, утилиту, API), опишите, как это делать._

**Пример:**
> Чтобы добавить аутентификацию на новую страницу, оберните ее в HOC `withAuth`:
> ```javascript
> import { withAuth } from '@/src/core/auth';
>
> const MyProtectedPage = () => <div>Секретный контент</div>;
>
> export default withAuth(MyProtectedPage);
> ```
