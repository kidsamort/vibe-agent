## 3. План Реализации

_**Инструкция:** Это самая важная часть. Опишите пошаговый план действий. Будьте технически точны: укажите названия функций, структуру данных, логику алгоритмов._

### Шаг 1: [Название шага, например, "UI и Frontend-логика"]
-   **Компонент `ForgotPasswordScreen`**: Будет содержать одно поле `email` и кнопку "Отправить".
-   **Состояние:** Будет использовать `useState` для управления вводом и состоянием загрузки/ошибки.
-   **Действие:** При отправке формы будет вызываться метод `passwordService.sendResetLink(email)`.

### Шаг 2: [Название шага, например, "Бэкенд и API"]
-   **Эндпоинт `/forgot-password`**:
    1.  Принимает `email`.
    2.  Генерирует уникальный, временный токен.
    3.  Сохраняет токен в базу данных, связывая его с пользователем.
    4.  Отправляет пользователю на email ссылку вида `[app_url]/reset-password?token=[token]`.

### Шаг 3: [Название шага, например, "Завершение процесса"]
-   **Страница `/reset-password`**:
    1.  Получает `token` из URL.
    2.  Проверяет валидность токена на бэкенде.
    3.  Если токен валиден, отображает форму для ввода нового пароля.
    4.  Отправляет новый пароль и токен на эндпоинт `/update-password`.
