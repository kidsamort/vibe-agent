# Руководство по Развертыванию

_**Цель этого документа:** Описать стандартизированный, пошаговый процесс развертывания (деплоя) новой версии приложения на **Staging** и **Production** окружения._

---

## 1. Подготовка к Деплою (Чек-лист)

Перед началом процесса развертывания убедитесь, что все пункты выполнены:

-   [ ] **Ветка `develop` обновлена:** `git pull origin develop`.
-   [ ] **Все тесты пройдены:** `pnpm test` завершается успешно.
-   [ ] **Сборка проекта проходит без ошибок:** `pnpm build` завершается успешно.
-   [ ] **Версия в `package.json` обновлена** в соответствии с семантическим версионированием (SemVer).
-   [ ] **Changelog обновлен:** В файле `CHANGELOG.md` описаны все ключевые изменения.
-   [ ] **Получено одобрение** от тимлида или QA-инженера.

---

## 2. Деплой на Staging

_Staging-окружение используется для финального тестирования перед релизом._

1.  **Создайте релизную ветку:**
    -   Из ветки `develop` создайте ветку `release/vX.Y.Z`, где `X.Y.Z` — новая версия.
    ```bash
    git checkout develop
    git pull
    git checkout -b release/v1.2.3
    ```

2.  **Запустите скрипт деплоя на Staging:**
    ```bash
    pnpm deploy:staging
    ```
    _Этот скрипт должен автоматически выполнить сборку, отправить артефакты на staging-сервер и перезапустить сервис._

3.  **Проведите регрессионное тестирование** на staging-окружении.

---

## 3. Деплой на Production

_**Внимание:** Деплой на Production — ответственная операция. Выполняйте ее только после успешного тестирования на Staging._

1.  **Слияние в `main`:**
    -   Создайте Pull Request из `release/vX.Y.Z` в `main`.
    -   Дождитесь одобрения и выполните слияние.
    -   Переключитесь на `main` и получите последние изменения:
        ```bash
        git checkout main
        git pull
        ```

2.  **Создайте тег в Git:**
    ```bash
    git tag -a v1.2.3 -m "Release version 1.2.3"
    git push origin v1.2.3
    ```

3.  **Запустите скрипт деплоя на Production:**
    ```bash
    pnpm deploy:prod
    ```

4.  **Проверьте работоспособность** основной функциональности на production-окружении.
